01 · Verweise · SVERWEIS
Der Klassiker

SVERWEIS — vier Argumente, eine Antwort.

1 · Suchwert

Wonach?

Die Artikelnummer aus der Bestellzeile — A2.

2 · Matrix

Wo?

Der Katalog. Mit $ fixiert, sonst wandert er beim Ausfüllen mit.

3 · Spaltenindex

Welche Spalte?

Gezählt ab der Suchspalte: Einzelpreis = Spalte 4 der Matrix.

4 · Genauigkeit

Exakt?

FALSCH = exakte Suche. Praktisch immer das, was Sie wollen.

Trainer-Notiz
Geplanter Fehler in 1A

Beim gemeinsamen Bauen die Matrix ohne $ lassen und runterziehen — ab Zeile ~20 kommen #NV. Gemeinsam mit F4 reparieren. Zweiter Klassiker: „Bei mir steht WAHR und es geht doch" → zeigen, was ungefähre Suche bei unsortierten Daten anrichtet: stiller Falschwert statt Fehler.

Trainer-Notiz
Kernargument

Das stärkste Argument kommt in Übung 1B selbst ans Licht: Spalte im Katalog einfügen → SVERWEIS liefert still die falsche Spalte, XVERWEIS bleibt korrekt. Das Experiment nicht vorwegnehmen.

01 · Verweise · Lösungsskizze

Mögliche Lösung.

1B · Beide Wege nebeneinander

' Bezeichnung klassisch

' Bezeichnung modern, mit Auffangwert
=XVERWEIS(A2; Artikel!$A$2:$A$500;
          Artikel!$B$2:$B$500; "nicht gefunden")

1C · Linksverweis

' INDEX/VERGLEICH
=INDEX(Artikel!$A$2:$A$500;
       VERGLEICH(H2; Artikel!$B$2:$B$500; 0))

' XVERWEIS — Richtung egal
=XVERWEIS(H2; Artikel!$B$2:$B$500;
          Artikel!$A$2:$A$500)

02 · Bedingungen · WENNS
Klassifizieren

WENNS nimmt den ersten Treffer.

  • Bedingungen werden von oben nach unten geprüft — der erste Treffer gewinnt.
  • Deshalb: immer von groß nach klein sortieren.
  • WAHR; "C-Kunde" fängt alles Übrige — das „Sonst".
Der Denkfehler

Wer mit B2>=5000 beginnt, hat plötzlich keine A-Kunden mehr: Jeder Großkunde ist ja auch ≥ 5000 — erster Treffer, fertig.

Altbau-Variante

Vor Excel 2019: verschachteltes WENN(WENN(…)). Gleiche Logik, mehr Klammern.

Trainer-Notiz
Geplanter Fehler in 2A

Mentale Landkarte

Vier Zonen. Das ist die ganze Pivot.

Was berechnen?

Betrag als Summe, Mittelwert, Anzahl — pro Schnittpunkt.

Filter

Was ausblenden?

Vorfilter über allem — später ersetzt durch klickbare Slicer.

05 · Datenqualität · Bedingte Formatierung
Sichtbar machen

Rot, wenn offen — die ganze Zeile.

  • Einfache Zellregeln („Textinhalt = offen") färben nur die eine Zelle.
  • Für ganze Zeilen: Formelregel mit gemischtem Bezug — $D2, nicht $D$2.
  • Vorher den ganzen Datenbereich ab Zeile 2 markieren — die Formel gilt relativ zur aktiven Zelle.
  • Mehr als rot/grün: Top-N-Regeln, Datenbalken, Farbskalen — verwaltet über „Regeln verwalten".
$D2 vs. $D$2

Mit $D$2 reagieren alle Zeilen auf Zeile 2. Das Dollarzeichen vor der Zeile festnageln heißt: jede Zeile schaut auf dieselbe Zelle. Der gemischte Bezug in Aktion.

Aufräumen

„Regeln verwalten" zeigt Reihenfolge, „Anhalten"-Häkchen und erlaubt gezieltes Löschen — wer hier sauber arbeitet, produziert keine Formatierungs-Friedhöfe.

Trainer-Notiz
Geplanter Fehler in 5A

Einmal absichtlich $D$2 schreiben → alle Zeilen reagieren auf Zeile 2. Gemeinsam analysieren. Und: Vor dem Anlegen der Regel laut ansagen, dass der ganze Bereich markiert sein muss — häufigste Fehlerquelle in 5B.

05 · Datenqualität · Datenvalidierung
Absichern

Falscheingaben gar nicht erst zulassen.

Liste

Dropdown

Statusspalte: nur „offen / in Arbeit / erledigt". Tippfehler wie „offfen" sind damit Geschichte.

Grenzen

Ganze Zahl 1–999

Mengenspalte: keine 0, keine 12,5, kein „viele". Zahlentyp und Bereich in einem Dialog.

Meldungen

Eigene Texte

Eingabemeldung als Tooltip, eigene Fehlermeldung beim Abweisen — die Mappe erklärt sich selbst.

Wichtige Grenze

Validierung wirkt nur auf neue Eingaben — bestehende Falschwerte bleiben. Dafür: Daten → Datenüberprüfung → „Ungültige Daten einkreisen".

Zwei Schleifen reichen

For, wenn Sie zählen. Do While, wenn Sie suchen.

For — Anzahl bekannt

For i = 2 To 100

Next i

Do While — bis die Daten enden

        Cells(r, 2).Value * Cells(r, 3).Value

' Im Code der UserForm "ReportForm":
Private Sub UserForm_Initialize()
    ' Liste füllen, sobald die Maske öffnet
    cboRegion.List = Array("Nord", "Süd", "Ost", "West")
End Sub

Private Sub cmdStart_Click()
    If cboRegion.Value = "" Then
        MsgBox "Bitte eine Region wählen."

    Unload Me                       ' Maske schließen
End Sub
Wiederverwendungs-Moment

Die Maske ruft dieselbe ReportErstellen-Sub aus Set 12 auf — jetzt mit der Region als Argument statt fest „Süd".

Drei Bausteine

  • Verhalten: Doppelklick auf den Knopf öffnet sein Click-Ereignis — dort steht der Code.
  • Aufruf: ReportForm.Show aus einem normalen Modul (oder vom Knopf auf dem Blatt).

Kür · Events — Code ohne KnopfDemo
Excel ruft den Code von selbst

Code, der von selbst reagiert.

' Im Modul "DieseArbeitsmappe":
Private Sub Workbook_Open()
    MsgBox "Willkommen — heute ist der " & Date
End Sub

' Im Blatt-Modul "Rohdaten":
Private Sub Worksheet_Change(ByVal Target As Range)
    If Target.Column = 4 Then   ' Betrag geändert
        Application.EnableEvents = False
        Cells(Target.Row, 5).Value = Now
        Application.EnableEvents = True
    End If
End Sub

Zwei typische Events

EreignisLäuft, wenn …
Workbook_Opendie Mappe geöffnet wird
Worksheet_Changeeine Zelle geändert wird
Die eine Falle

Schreibt der Code selbst in eine Zelle, löst er das Change-Event erneut aus — Endlos-Schleife. Application.EnableEvents = False verhindert das; danach wieder True.

Trainer-Notiz
Wo der Code wohnt

Wichtigster Punkt: Event-Prozeduren liegen nicht im normalen Modul, sondern im Objekt-Modul (Blatt bzw. „DieseArbeitsmappe"). Im Editor oben links das Objekt, oben rechts das Ereignis wählen — dann schreibt VBA den Kopf selbst.

Breakout · Paararbeit

Lassen Sie eine Zelle automatisch einen Zeitstempel bekommen, sobald daneben ein Betrag eingetragen wird — ohne Knopf.

  • Schritt 1: Im Blatt-Modul „Rohdaten" das Worksheet_Change-Ereignis wählen.
  • Schritt 2: Nur reagieren, wenn die geänderte Zelle in der Betrags-Spalte liegt.

Bleibt EnableEvents nach einem Absturz auf False, reagiert nichts mehr. Einmal Application.EnableEvents = True im Direktfenster ausführen — und künftig On Error nutzen, um es zurückzusetzen.
